One file. openai + numpy, no vector-DB server. Every tutorial ships answer(). This one ships evaluate() — because recall@k is the only number that tells you if your next change helped.

The pipeline, end to end

Data flows left→right. The dashed loop is the part everyone skips: evaluate() measures retrieval, hands you the chunks that failed, and you fix those — not vibes.

This animated diagram is the walkthrough — editable SVG, not a video file. Open the page and watch the data move.

Quickstart — 4 lines, evaluate first

copy
# pip install openai numpy   •   export OPENAI_API_KEY=sk-...
from rag import RAG

r = RAG().add(open("docs.txt").read().split("\n\n"))
print(r.evaluate(k=5))          # ← ALWAYS start here: recall@k, mrr, failures
print(r.answer("your question"))

evaluate() generates a question from each chunk and checks whether that chunk comes back for its own question. Low recall@k? Your retrieval is broken and no prompt-tuning will save you. Read result["failures"] — those chunk indices are your entire to-do list.

Upgrades — add each only when recall@k says so

1 · Persist
Corpus outlives the process. Exact search, zero infra.
2 · Rerank
LLM reorders the top-20. The usual next win once recall caps.
3 · Route
Send each query to the right collection. Separate routing misses from retrieval misses.

1 · Persist — the honest "vector DB"

numpy cosine is exact to ~100k chunks; save() makes it durable. Swap in pgvector/LanceDB inside save()/load() when the matrix stops fitting in RAM — not before.

copy
r.save("store")               # → store.npz + store.json
r = RAG.load("store")         # next run: no re-embedding, no API cost

2 · Rerank — and prove it helped

copy
base = r.evaluate(k=5)
lift = r.evaluate(k=5, rerank=True)   # pull 20 by cosine, LLM reorders
print(base["recall@k"], "→", lift["recall@k"])   # ship it ONLY if this goes up

r.answer("your question", rerank=True)

3 · Route across collections

copy
from rag import RAG, Router

billing = RAG("billing", "invoices, refunds, payment errors").add(billing_docs)
api     = RAG("api", "endpoints, auth, rate limits").add(api_docs)

bot = Router([billing, api])
bot.answer("why was my card declined?")   # → routed to billing, then answered
